# ТЗ: LINKEON.IO — функциональная и идеологическая часть

---

## ЧАСТЬ I. ИДЕОЛОГИЯ

### 1.1. Миссия

**LINKEON.IO — это нейросеть для роста и развития бизнеса и человека.**

Платформа объединяет в одном окне три вещи, которые сегодня живут в разных сервисах:

1. **Экспертный совет.** Доступ к команде ИИ-специалистов (юрист, бухгалтер, маркетолог, HR, психолог, коуч, бизнес-наставник) 24/7, без записи и без счёта на 5–15 тысяч за консультацию.
2. **Нетворкинг по смыслам.** Поиск нужных людей не по «образованию и опыту работы», как в LinkedIn, а по реальным **ценностям, целям и намерениям** — тому, что определяет, сработаетесь ли вы.
3. **Производство контента.** Генерация текстов, изображений и видео для бизнеса прямо в диалоге, без переключения между инструментами.

### 1.2. Для кого

- **Предприниматели и фрилансеры** — закрывают функции отсутствующих в штате юриста / бухгалтера / маркетолога, ищут партнёров и подрядчиков.
- **Профессионалы в личностном росте** — коучи, психологи, эксперты — используют как «второе мнение» и для поиска клиентов.
- **Люди в точке выбора** — те, кто ищет окружение «по духу»: создать семью, сменить профессию, найти команду в проект, переехать.

### 1.3. Принципы продукта (то, чем мы отличаемся)

| Принцип | Что это значит на практике |
|---|---|
| **Профиль складывается сам.** | Пользователь не заполняет анкеты. Его «ценности / интересы / навыки / желания / намерения» извлекаются из обычных разговоров с ассистентами и хранятся в графовой модели. Чем больше пользуешься — тем точнее тебя видит система. |
| **Поиск по смыслу, а не по ключевым словам.** | «Ищу со-основателя для экопроекта в Сочи» возвращает не людей со словом «эко» в био, а тех, у кого совпадают ценности и комплементарные навыки. |
| **Согласие перед знакомством.** | Никаких холодных контактов и утечки телефонов. Связь — только через запросы на общение с явным «Принять/Отклонить». Это превентивная защита от спама и харрасмента. |
| **Динамическая цена, а не подписка.** | Платишь за результат, а не за «безлимит, которым ты не воспользуешься». Простой вопрос стоит дёшево, сложная задача — дороже. Счётчик после каждого действия. |
| **AI первым, человек — когда нужно.** | В поддержке, в продаже, в обучении — ИИ закрывает 90% обращений мгновенно. Команда подключается только на сложных и эмоциональных кейсах. |
| **Один разговор — много задач.** | Из чата с ассистентом можно тут же сгенерировать картинку, оживить её в видео, прикрепить PDF, найти под задачу подрядчика, проверить совместимость. Не нужно «выходить и открыть другой сервис». |
| **Партнёр получает долю.** | Многоуровневая партнёрская программа: тот, кто привёл пользователя, получает % с его расходов всё время, пока тот пользуется платформой. |

### 1.4. Тон и язык

- Обращение **на «вы»** в UI и в ответах ассистентов (если пользователь сам не перешёл на «ты»).
- Без канцелярита и без «коучингового» сюсюканья. Прямо, по делу, с уважением.
- Без эмодзи в системных текстах. В сообщениях ассистентов — только если уместно по контексту.
- Тексты ошибок — человеческие: «Не нашли вас в базе. Проверьте номер?», а не «Error 404».

### 1.5. Метрики, которым мы служим

1. **Глубина профиля** — сколько ценностей / целей / навыков извлечено и подтверждено пользователем.
2. **Качество матчинга** — % принятых запросов на общение из числа отправленных.
3. **Время до первой пользы** — от регистрации до первого осмысленного результата (ответ юриста / найденный матч / готовая картинка).
4. **AI-share поддержки** — % тикетов, закрытых без участия команды.
5. **LTV / стоимость токена** — экономика динамической цены.

Если решение в продукте ухудшает любую из этих пяти метрик — оно неправильное, даже если красиво выглядит.

## ЧАСТЬ II. ФУНКЦИОНАЛ

Платформа состоит из **девяти функциональных блоков**. Каждый блок описан как: *что это / какие сценарии / какие правила и ограничения*.

### 2.1. Регистрация и согласия

**Что.** Вход по номеру телефона + SMS-код. Никаких email/паролей.

**Сценарий.**
1. Пользователь вводит телефон в международном формате.
2. Получает 6-значный код. Может вставить его из буфера обмена одной кнопкой.
3. Подтверждает согласие: «Мне больше 18 лет, принимаю Пользовательское соглашение, Политику конфиденциальности, согласен на обработку персональных данных».
4. Видит модал «Описание услуг и порядок оплаты» (требование банка-эквайера).
5. Попадает на главный экран с начисленными **25 000 стартовых токенов**.

**Правила.**
- Без согласия на 18+ и ПД — внутрь не пускаем.
- Повторная отправка SMS — с таймером (защита от перебора).
- Удаление аккаунта — в один клик из профиля, удаляет все данные.

### 2.2. Чат с ИИ-ассистентами (главный экран)

**Что.** Каталог из ~15 ассистентов в трёх категориях + персональный ассистент «Роман».

**Категории и роли.**
- **Для бизнеса:** HR, бухгалтер, юрист, маркетолог, бизнес-наставник, копирайтер.
- **Личностный рост:** коуч, психолог, игропрактик, нумеролог, ассистент по осознанности, астролог, Human Design.
- **Личный ассистент Роман** — универсальный помощник «по всему».

**Сценарий.**
1. Пользователь выбирает ассистента в левой панели (или снизу — на мобиле).
2. Пишет вопрос текстом, надиктовывает голосом или прикладывает PDF.
3. Ответ приходит **потоком** (печатается на глазах), без ожидания.
4. В ответе ассистента могут быть **кликабельные кнопки и ссылки** («Записать в профиль», «Найти подходящих людей», «Сгенерировать картинку», «Открыть Совместимость»).
5. История диалога сохраняется per-ассистент. При переключении ассистента — открывается прошлый контекст.
6. Под каждым сообщением — сколько токенов списано.

**Правила.**
- Один и тот же пользователь в разных вкладках — синхронизирован (сменил ассистента → во всех вкладках сменился).
- При балансе < 1000 токенов — баннер «Пополнить».
- В чате можно перегенерировать ответ и очистить историю (с подтверждением).
- Голосовой ввод — там, где браузер поддерживает; иначе кнопка скрыта.
- Загрузка PDF — со всплывающим окном «Что сделать с этим файлом? (проанализируй / переведи / извлеки ключевое...)».

### 2.3. Профиль пользователя (графовый, автоформируемый)

**Что.** Не анкета, а живой портрет, который собирается ИИ из реплик пользователя.

**Что хранится.**
- **Ценности** (что важно).
- **Убеждения** (как смотрит на мир).
- **Желания** (что хочется получить).
- **Намерения** (что готов делать).
- **Интересы** (чем увлекается).
- **Навыки** (что умеет).
- Для каждого пункта — **уровень уверенности** (ИИ может ошибиться) и **timeline** (как менялось со временем).

**Что редактируется руками.**
- Имя, фамилия, аватар.
- Настройки приватности: кто видит профиль (все / только совпадения / никто), показывать ли ценности, кто может писать в чат (все / совпадения / никто).

**Сценарий.**
- Чем больше пользователь общается с ассистентами, тем плотнее профиль.
- Пользователь видит профиль и может **подтвердить или удалить** любую сущность (борьба с галлюцинациями ИИ).
- Профиль участвует в поиске и совместимости — но только в той мере, в которой разрешено настройками приватности.

### 2.4. Нетворкинг (поиск людей и совместимость)

Раздел из четырёх вкладок.

#### 2.4.1. Поиск людей
- Поле ввода в свободной форме: «Ищу со-основателя для эко-поселка», «Нужна психологическая консультация», «Хочу научиться играть на гитаре», «Создаю фитнес-центр, нужен партнёр».
- ИИ возвращает ранжированные карточки: **% совпадения**, общие ценности, комплементарные цели, фото и краткое описание.
- Действия на карточке: открыть профиль, проверить совместимость, **отправить запрос на общение**.

#### 2.4.2. Совместимость
- Пользователь добавляет один или несколько телефонов (вручную или импортом из контактов телефона, если браузер поддерживает).
- ИИ анализирует пары профилей и выдаёт **развёрнутый разбор**: совпадения и расхождения по ценностям, интересам, навыкам, потенциальные сильные и слабые стороны взаимодействия.
- Можно запустить совместимость прямо из карточки запроса или открытого чата с собеседником.

#### 2.4.3. Запросы (Contact Requests)
- Входящие и исходящие приглашения к общению.
- К запросу прикладывается **intro-сообщение до 500 символов**.
- На входящем запросе: «Посмотреть профиль», «Проверить совместимость», **«Принять»**, **«Отклонить»**.
- Отклонение **не уведомляет** отправителя (без сцен).
- Лимиты: не более **10 активных исходящих** одновременно, не более **5 новых в час** (анти-спам).
- Если получатель сам ранее отправил запрос — повторная отправка автоматически **принимает встречный запрос** и сразу открывает чат.

#### 2.4.4. Чаты между пользователями
- Появляются только после принятого запроса.
- Текст до 4000 символов, отметки прочтения, индикатор «печатает».
- Аватар и имя в шапке кликабельны → полный профиль.
- Меню «три точки»: посмотреть профиль, проверить совместимость, **пожаловаться**, **заблокировать**.
- Жалоба уходит в модерацию. Блокировка двусторонняя: ни писать, ни получать запросы от блокированного нельзя.
- Обмен личными контактами (телефон, мессенджеры) — только по обоюдному решению, прямо в переписке. Платформа не раскрывает номера сама.

### 2.5. Генерация изображений

**Что.** Создание картинок по текстовому описанию.

**Сценарий.**
- Из чата ассистента: «нарисуй обложку для презентации про эко-фермы», «создай иллюстрацию к посту».
- Из отдельного раздела «Картинки» — поле ввода + кнопка «Сгенерировать».
- Готовая картинка появляется в ленте чата или в галерее раздела.
- Можно скачать.

**Правила.**
- Стоимость в токенах — динамическая, отображается до запуска.
- Контент-модерация на запрещённые темы (на стороне модели).

### 2.6. Генерация видео

**Что.** Четыре режима генерации:

1. **Текст → Видео** — описать сцену словами.
2. **Картинка → Видео** — оживить статичное изображение.
3. **Продолжение** — добавить +5 секунд к ранее созданному ролику.
4. **Липсинк** — наложить голос/аудио на видео, синхронизируя движение губ с лицом.

**Параметры.**
- Модель: Standard (быстрее, дешевле) / Premium (качественнее).
- Качество: Обычное / Про.
- Длительность: 5 или 10 секунд.
- Дополнительно: негативный промпт, CFG Scale, тип камеры, исходное изображение/видео/аудио.

**Сценарий.**
- Стоимость пересчитывается на лету в зависимости от выбранных параметров и показывается на кнопке «Создать».
- После запуска — job-карточка со статусом: «В очереди» → «Генерируется (с таймером)» → «Готово»/«Ошибка». Типичное время — 3–5 минут.
- Раздел «Мои видео» — галерея всех созданных роликов с действиями: смотреть, скачать, отправить в чат, продолжить, применить липсинк, удалить.

**Правила.**
- Лимит длительности — 10 секунд за один запрос.
- Липсинк — только на Standard-модели (особенность движка).

### 2.7. Дозвон (исходящие звонки ассистента)

**Что.** Ассистент сам звонит пользователю на телефон и проводит сессию по заданному сценарию.

**Сценарий.**
- Пользователь выбирает сценарий (коучинговая сессия, утренняя планёрка, «разбор полётов» и т.п.) и время.
- В назначенное время приходит звонок. Ассистент говорит, слушает, делает выводы.
- После звонка краткий протокол попадает в чат и/или обновляет профиль.

### 2.8. Токены, оплата, партнёрская программа

#### 2.8.1. Токены — внутренняя валюта
- Стартовый бонус новому пользователю: **25 000 токенов**.
- Расходуются на: сообщения ассистенту, генерации картинок и видео, поиск людей, анализ совместимости, дозвоны.
- **Динамическая цена.** Не фиксированный «тариф за сообщение», а реальная стоимость задачи — как Uber подстраивает цену под спрос, расстояние и время.
- Счётчик после каждой операции, баланс в боковой панели обновляется каждые несколько секунд.

#### 2.8.2. Покупка
- Пакеты от 149 ₽ до 1990 ₽ (Basic / Extended / Professional).
- Оплата — банковская карта через YooKassa.
- После оплаты автоматический возврат в чат, токены зачисляются мгновенно.
- Объяснение «как считается цена» открывается в один клик.

#### 2.8.3. Промокоды
- Поле ввода кода → бонусные токены на баланс.
- Коды выпускаются админами (партнёрские акции, компенсации).

#### 2.8.4. Партнёрская программа (реферал)
- Партнёр получает **многоуровневую комиссию** с расходов приведённых им пользователей.
- Два слоя: **прямые** (первый уровень) и **апстрим** (вторые/третьи уровни — для тех, кто привёл партнёра).
- В кабинете партнёра: реферальная ссылка (с копированием в один клик), сколько людей привёл, сколько они потратили, текущая комиссия, что выплачено, что в ожидании, помесячная история.
- Подключение — только через администратора (нет публичной регистрации в программу; защита от фрода).

### 2.9. Поддержка пользователей

**Модель — гибридная: AI отвечает первым, команда подключается по эскалации.**

**Сценарий пользователя.**
1. Открывает «Поддержка» — это обычный чат.
2. Пишет вопрос. AI-поддержка отвечает первой: знает продукт, проверяет статус сервисов и статус генераций пользователя, может посмотреть его историю операций.
3. Если AI понимает, что вопрос сложный или эмоциональный — **передаёт команде**. В чате появляется баннер «Передано команде».
4. Команда отвечает в **тот же чат** — пользователь не переключается никуда.
5. По решению вопроса — «Закрыто». Если возник новый — кнопка «Открыть новый тикет».

**Сценарий команды (админка → вкладка Поддержка).**
- Лента тикетов с фильтрами: **Требуют внимания / В работе / AI ведёт / Решены / Все**.
- KPI-шапка: сколько в каждом статусе, сколько новых за период, сколько решено, **% тикетов, которые AI закрыл без команды**, среднее время первого ответа, количество рефандов и сумма.
- В тикете: вся переписка, внутренние заметки (видны только команде), кнопки **«Решено»**, **«Вернуть AI»** (если эскалация была преждевременной).

### 2.10. Админка

Закрытая зона для пользователей с флагом администратора. Вкладки:

1. **Поддержка** — описано выше (2.9).
2. **Пользователи** — список + детальная карточка («Активность пользователя»): баланс, списано за период, сколько было запросов / картинок / видео / звонков, графики «токены и запросы по дням» с переключением периода, разбивка по ассистентам, последние транзакции, последние сообщения, история платежей.
3. **Платежи** — реестр операций.
4. **Токены** — ручные корректировки баланса (служебные начисления, компенсации).
5. **Использование** — аналитика продукта (когортная, по фичам).
6. **Ассистенты** — управление каталогом: название, категория, аватар, конфигурация поведения, активность.
7. **Купоны** — создание промокодов (номинал, лимит активаций, срок).
8. **Рефералы** — настройка партнёрской иерархии: кто чей апстрим, проценты по уровням, статистика.

### 2.11. Сквозные UX-требования

- **Языки:** русский (основной) и английский. Переключение в настройках.
- **Mobile-first.** На мобиле — нижняя навигация, на десктопе — боковая. Все экраны полноценно работают и на телефоне, и на компьютере.
- **Тосты** для системных уведомлений: «Скопировано», «Видео готово», «Профиль сохранён», «Недостаточно токенов».
- **MaintenancePage** — служебная страница «Сервис временно недоступен» с обратным отсчётом, включается одним флагом.
- **Помощь (`/help`)** — встроенный FAQ и краткая инструкция по всем разделам. Доступна всегда, в том числе из меню.
- **Визитка** — публичная страница пользователя для шеринга наружу.
- **Соцсети** — раздел подключения внешних аккаунтов (сейчас Telegram, на вырост — другие).

## ЧАСТЬ III. ЧЕГО В ПРОДУКТЕ БЫТЬ НЕ ДОЛЖНО

Чтобы не уйти в обобщённость, явно фиксируем границы:

- **Нет ленты и подписок.** Это не соцсеть. Контент создаётся не для аудитории, а для дела.
- **Нет публичного рейтинга и лайков.** Совместимость — между двумя людьми, без публичного скоринга.
- **Нет поиска по «полу, возрасту, гео».** Только по смыслу. Это не дейтинг.
- **Нет холодных контактов.** Прямой связи между незнакомыми пользователями не существует — только через запросы.
- **Нет «безлимитной подписки».** Только токены и динамическая цена.
- **Нет анкет и опросников.** Профиль собирается из разговора, а не заполнением 30 полей.
- **Нет тёмной темы** (пока — приоритет ниже).
